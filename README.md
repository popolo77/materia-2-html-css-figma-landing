# README
## Materia 2 - HTML, CSS, Responsive Design y Figma Handoff

### Programa Complementario de Formación Frontend - UTN

Repositorio académico-profesional correspondiente a la Materia 2: HTML, CSS, Responsive Design y Figma Handoff, dentro del Programa Complementario de Formación Frontend diseñado para complementar la Tecnicatura Universitaria en Programación de la UTN[cite: 1].

**Objetivo:** Construir interfaces web profesionales, interactivas, accesibles y totalmente responsivas a partir de maquetas de diseño en Figma, utilizando HTML5 semántico y CSS moderno, consolidando el flujo de desarrollo y el despliegue progresivo[cite: 1].

### Programa oficial de la materia

Estructura de contenidos organizados por unidades temáticas para un total de 48 horas sugeridas[cite: 1]:

* **Unidad 1: HTML semántico:** estructura documental, etiquetas semánticas, formularios, tablas, metadatos, accesibilidad inicial y SEO básico[cite: 1].
* **Unidad 2: CSS profesional:** selectores, cascada, especificidad, box model, tipografías, variables CSS, organización de estilos[cite: 1].
* **Unidad 3: Layout y responsive:** Flexbox, Grid, media queries, mobile first, patrones de layout y componentes reutilizables[cite: 1].
* **Unidad 4: Design Handoff con Figma:** lectura de maquetas, medición de espaciados, tipografías, colores, componentes, prototipos, exportación de assets y traducción fiel a HTML/CSS[cite: 1].
* **Unidad 5: Accesibilidad y performance inicial:** contraste, navegación por teclado, textos alternativos, optimización de imágenes y primer reporte Lighthouse[cite: 1].
* **Unidad 6: Deploy inicial:** publicación de landing en Netlify o Vercel con URL pública y README de entrega[cite: 1].

### Contenidos trabajados hasta la fecha

Durante el inicio de esta materia se estructuraron los cimientos de la landing page de Uber Tracker mediante los siguientes conceptos técnicos[cite: 1]:
* Estructura básica y obligatoria de un documento HTML5 (html, head, body)[cite: 1].
* Arquitectura de la información mediante etiquetas semánticas estructurales (header, main, nav, section, footer)[cite: 1].
* Jerarquía de contenidos con títulos (h1 a h6), párrafos, listas y enlaces adaptados al ecosistema del conductor[cite: 1].
* Elementos de formularios avanzados para la captura de datos (form, input, label, textarea, select, button)[cite: 1].
* Criterio de accesibilidad web inicial mediante la vinculación unívoca entre texto y control (for / id)[cite: 1].
* Organización de información financiera compleja utilizando tablas estructuradas (table, thead, tbody, tr, th, td)[cite: 1].
* Configuración del bloque head con metadatos técnicos (description, keywords, author) orientados a SEO básico[cite: 1].
