# README
## Materia 2 - HTML, CSS, Responsive Design y Figma Handoff

### Programa Complementario de Formación Frontend - UTN

Repositorio académico-profesional correspondiente a la Materia 2: HTML, CSS, Responsive Design y Figma Handoff, dentro del Programa Complementario de Formación Frontend diseñado para complementar la Tecnicatura Universitaria en Programación de la UTN.

**Objetivo:** Construir interfaces web profesionales, interactivas, accesibles y totalmente responsivas a partir de maquetas de diseño en Figma, utilizando HTML5 semántico y CSS moderno, consolidando el flujo de desarrollo y el despliegue progresivo.

### Programa oficial de la materia

Estructura de contenidos organizados por unidades temáticas para un total de 48 horas sugeridas:

* **Unidad 1: HTML semántico:** estructura documental, etiquetas semánticas, formularios, tablas, metadatos, accesibilidad inicial y SEO básico.
* **Unidad 2: CSS profesional:** selectores, cascada, especificidad, box model, tipografías, variables CSS, organización de estilos.
* **Unidad 3: Layout y responsive:** Flexbox, Grid, media queries, mobile first, patrones de layout y componentes reutilizables.
* **Unidad 4: Design Handoff con Figma:** lectura de maquetas, medición de espaciados, tipografías, colores, componentes, prototipos, exportación de assets y traducción fiel a HTML/CSS.
* **Unidad 5: Accesibilidad y performance inicial:** contraste, navegación por teclado, textos alternativos, optimización de imágenes y primer reporte Lighthouse.
* **Unidad 6: Deploy inicial:** publicación de landing en Netlify o Vercel con URL pública y README de entrega.

### Contenidos trabajados hasta la fecha

Durante esta materia se estructuraron los cimientos de la landing page de Uber Tracker mediante los siguientes conceptos técnicos:
* Estructura básica y obligatoria de un documento HTML5 (html, head, body).
* Arquitectura de la información mediante etiquetas semánticas estructurales (header, main, nav, section, footer).
* Jerarquía de contenidos con títulos (h1 a h6), párrafos, listas y enlaces adaptados al ecosistema del conductor.
* Elementos de formularios avanzados para la captura de datos (form, input, label, textarea, select, button).
* Criterio de accesibilidad web inicial mediante la vinculación unívoca entre texto y control (for / id).
* Organización de información financiera compleja utilizando tablas estructuradas (table, thead, tbody, tr, th, td).
* Configuración del bloque head con metadatos técnicos (description, keywords, author) orientados a SEO básico.
* Enlace y configuración de hojas de estilo externas (link rel="stylesheet").
* Centralización del diseño mediante Variables CSS globales (:root) para consistencia de marca.
* Control del espaciado, comportamiento y desbordamiento de contenedores utilizando las propiedades del Box Model (margin, padding, max-width, box-sizing).
* Implementación de selectores de etiquetas, selectores compuestos y pseudo-clases (hover, focus) para mejorar la experiencia de usuario.
